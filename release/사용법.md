# 파일 자동 격리/삭제 프로그램 사용법

PC에 쌓이는 오래된 파일을 자동으로 감지해 격리 폴더로 옮기고,
일정 기간이 지나면 영구 삭제해 용량을 관리합니다.

> **GitHub 저장소:** https://github.com/Sdapaul/delete_pii
> - 소스코드: 저장소 루트 (`file_cleaner.py`)
> - 배포 패키지 스크립트: `release/` 폴더 (`setup.bat`, `run_cleaner.bat`, `배포방법.md`)

---

## 파일 구성

### Python 버전 (직접 실행)

| 파일 | 역할 |
|---|---|
| `file_cleaner.py` | 메인 프로그램 |
| `config.json` | 설정 파일 (직접 편집) |
| `run_cleaner.bat` | 작업 스케줄러용 실행 래퍼 |
| `activity_log.txt` | 작업 결과 로그 (자동 생성) |

### EXE 배포 버전 (Python 불필요)

| 파일 | 역할 |
|---|---|
| `build.bat` | 배포자용 EXE 빌드 스크립트 (인터넷 PC에서 실행) |
| `release/setup.bat` | 수신자용 자동 설치 스크립트 |
| `release/run_cleaner.bat` | 스케줄러용 실행 래퍼 (EXE 버전) |
| `release/배포방법.md` | 배포 절차 안내 |

> `config.json`과 `activity_log.txt`는 `file_cleaner.py` (또는 `file_cleaner.exe`)와 **같은 폴더**에 생성됩니다.

## 사전 요구사항 — Python 설치

프로그램 실행에 Python 3.8 이상이 필요합니다.

설치 여부 확인:

`Win + R` → `cmd` 입력 → Enter:

```
python --version
```

`Python 3.x.x`가 출력되면 설치된 것입니다. 출력이 없거나 오류가 나면 [python.org](https://python.org)에서 설치하고, 설치 시 **"Add Python to PATH"** 체크박스를 반드시 선택하세요.

## 설치 및 설정

### STEP 1 — 파일 배치

`file_cleaner.py`, `config.json`, `run_cleaner.bat` 세 파일을 **같은 폴더**에 둡니다.

권장 경로 예시:

```
C:\FileCleaner\
  ├── file_cleaner.py
  ├── config.json
  └── run_cleaner.bat
```

---

### STEP 2 — 내 Windows 사용자명 확인

`config.json`에 본인 경로를 입력해야 합니다.

`Win + R` → `cmd` 입력 → Enter:

```
echo %USERNAME%
```

출력된 이름이 내 사용자명입니다. 이후 단계에서 `사용자명` 자리에 이 값을 넣습니다.

```
예시 출력: hong
→ 경로: C:\Users\hong\Downloads
```

---

### STEP 3 — config.json 설정

`config.json`을 메모장으로 열어 본인 환경에 맞게 수정합니다.

```json
{
    "target_path":     "C:\\Users\\사용자명\\Downloads",
    "extensions":      [".tmp", ".log"],
    "keywords":        [],
    "quarantine_days": 30,
    "delete_days":     7,
    "quarantine_dir":  "C:\\Users\\사용자명\\Downloads\\Quarantine",
    "runs_per_day":    0,
    "exclude_paths":   []
}
```

| 항목 | 설명 |
|---|---|
| `target_path` | 검사할 폴더. `사용자명`을 실제 Windows 로그인 이름으로 변경. `C:\\`로 설정하면 C 드라이브 전체 스캔 |
| `extensions` | 대상 확장자. 빈 배열 `[]`이면 전체 파일 대상 |
| `keywords` | 파일명 포함 단어. 빈 배열 `[]`이면 전체 파일 대상 |
| `quarantine_days` | 파일 생성 후 이 일수 **이상**이면 격리 폴더로 이동 |
| `delete_days` | 격리 폴더 이동 후 이 일수를 초과하면 영구 삭제 |
| `quarantine_dir` | 격리 폴더 경로. 없으면 자동 생성됨 |
| `runs_per_day` | `0` 고정. 실행 주기는 작업 스케줄러가 관리 |
| `exclude_paths` | 추가로 제외할 경로 목록. 빈 배열이면 적용 안 함 |

> 경로의 `\`는 반드시 `\\`로 두 번 입력해야 합니다.

**Ctrl+S** 로 저장 후 메모장을 닫습니다.

---

### 고급 설정 — C 드라이브 전체 대상으로 스캔하기

특정 폴더 대신 C 드라이브 전체를 검사 대상으로 설정할 수 있습니다.
`target_path`를 `C:\\`로 변경하기만 하면 됩니다.

```json
{
    "target_path":     "C:\\",
    "extensions":      [".tmp", ".log"],
    "keywords":        ["temp", "임시"],
    "quarantine_days": 60,
    "delete_days":     14,
    "quarantine_dir":  "C:\\Users\\사용자명\\Downloads\\Quarantine",
    "runs_per_day":    0,
    "exclude_paths":   []
}
```

#### 자동으로 보호되는 경로 (탐색하지 않음)

아래 경로들은 코드 내에서 탐색이 완전히 차단됩니다. 설정 없이도 보호됩니다.

| 보호 경로 | 이유 |
|---|---|
| `C:\Windows` | 운영체제 시스템 파일 |
| `C:\Program Files`, `C:\Program Files (x86)` | 프로그램 설치 파일 |
| `C:\ProgramData` | 전역 앱 데이터 |
| `C:\System Volume Information` | 복원 지점 데이터 |
| `C:\$Recycle.Bin` | 휴지통 |
| `C:\Recovery`, `C:\boot` | 복구·부팅 파일 |
| `%APPDATA%` (예: `C:\Users\사용자명\AppData\Roaming`) | 앱 설정·저장 데이터 |
| `%LOCALAPPDATA%` (예: `C:\Users\사용자명\AppData\Local`) | 앱 캐시·로컬 데이터 |

> `AppData`는 프로그램이 실행되는 환경에서 자동으로 탐지되어 보호 목록에 추가됩니다.

#### exclude_paths — 추가 제외 경로 지정

위 보호 목록에 없는 경로를 추가로 제외하려면 `exclude_paths`를 사용합니다.

```json
"exclude_paths": [
    "C:\\Users\\사용자명\\OneDrive",
    "C:\\MyApp",
    "C:\\Users\\사용자명\\Documents"
]
```

#### C 드라이브 전체 스캔 시 주의 사항

- `extensions` 또는 `keywords` 중 하나는 **반드시 지정**하세요. 두 항목이 모두 빈 배열이면 C 드라이브의 모든 오래된 파일이 격리 대상이 됩니다.
- `quarantine_days`를 충분히 길게 (60일 이상) 설정하고, 격리 결과를 며칠 관찰한 뒤 기준을 좁혀가는 것을 권장합니다.
- `C:\Users` 하위에는 여러 사용자 계정 폴더가 포함되므로, 다른 계정의 파일이 대상이 될 수 있습니다. 본인 계정만 대상으로 하려면 `target_path`를 `C:\\Users\\사용자명` 으로 지정하세요.

---

### 설치 경로 안내 (setup.bat)

`setup.bat`을 실행하면 설치 경로를 직접 입력할 수 있습니다.

```
설치 폴더를 입력하세요 [C:\FileCleaner]:
```

- **Enter만 누르면** 기본값 `C:\FileCleaner\`에 설치됩니다.
- **다른 경로를 입력하면** 그 경로에 설치됩니다. 예: `C:\ProgramData\FileCleaner`
- 설치 후 **작업 스케줄러가 그 경로의 프로그램을 실행**하므로, 설치 경로를 변경하면 스케줄러도 새로 등록해야 합니다.
- 이미 설치된 경로를 변경하려면 `setup.bat`을 다시 실행하면 기존 스케줄러를 덮어쓰고 새 경로로 재설치됩니다.

### STEP 4 — 동작 확인 (수동 실행)

**Python 버전:**

`Win + R` → 아래 명령어 입력 → Enter:

```
cmd /k python "C:\FileCleaner\file_cleaner.py" --config
```

> `C:\FileCleaner\` 부분은 실제 파일을 놓은 경로로 바꾸세요.

**EXE 배포 버전 (`setup.bat`으로 설치한 경우):**

`Win + R` → 아래 명령어 입력 → Enter:

```
cmd /k "C:\FileCleaner\file_cleaner.exe" --config
```

> 설치 시 기본 경로 `C:\FileCleaner\`가 아닌 다른 폴더를 지정했다면 그 경로로 변경하세요.

정상 실행 시 아래와 같이 출력됩니다.

```
  → C:\FileCleaner\config.json 로드 완료

[모드] 일회성 실행

2026-04-24 20:00:49  INFO  ========================================================
2026-04-24 20:00:49  INFO  실행 시작
2026-04-24 20:00:49  INFO    대상: C:\Users\hong\Downloads
2026-04-24 20:00:49  INFO    확장자: ['.tmp', '.log']
2026-04-24 20:00:49  INFO    키워드: (전체)
2026-04-24 20:00:49  INFO    격리 기준: 30일  /  삭제 기준: 7일
2026-04-24 20:00:49  INFO  ========================================================
2026-04-24 20:00:49  INFO  [격리 소계]  0개 파일 이동
2026-04-24 20:00:49  INFO  [삭제 소계]  0개 파일 영구 삭제
2026-04-24 20:00:49  INFO  실행 완료  —  격리 0개 / 삭제 0개
```

`격리 0개 / 삭제 0개`는 오류가 아닙니다. 조건에 해당하는 파일이 없을 때의 정상 결과입니다.

### STEP 5 — 작업 스케줄러 등록

매일 자동 실행되도록 Windows 작업 스케줄러에 등록합니다.

**1.** `Win + S` → **작업 스케줄러** 검색 → 실행

**2.** 우측 [작업 만들기] 클릭

**3.** [일반] 탭
- 이름: `File_Auto_Cleaner`
- **"사용자의 로그온 여부에 관계없이 실행"** 선택

**4.** [트리거] 탭 → [새로 만들기]
- 매일, 시작 시간: `09:00:00`
- [확인]

**5.** [동작] 탭 → [새로 만들기]
- 동작: **프로그램 시작**
- 프로그램/스크립트: `cmd`
- 인수 추가: `/c "C:\FileCleaner\run_cleaner.bat"`
- [확인]

> `C:\FileCleaner\run_cleaner.bat` 부분은 실제 경로로 변경하세요.

**6.** [확인] → 계정 암호 입력 후 등록 완료

---

### STEP 6 — 격리 폴더 확인

격리된 파일은 `config.json`의 `quarantine_dir`에 설정한 경로에 쌓입니다.

```
C:\Users\사용자명\Downloads\Quarantine
```

탐색기 주소창에 위 경로(사용자명 교체)를 입력하면 바로 열립니다.
이 폴더를 **주기적으로 확인**해 필요한 파일이 잘못 들어가지 않았는지 점검하세요.
`delete_days`(기본 7일)가 지나면 이 폴더의 파일은 자동으로 영구 삭제됩니다.

## 제거 방법

### 자동 제거 — uninstall.bat (권장)

`uninstall.bat` 은 어디서 실행해도 **제거할 설치 폴더를 직접 입력**받습니다.
설치 폴더가 `C:\FileCleaner` 가 아닌 `C:\ProgramData\FileCleaner` 등 다른 경로여도 동일하게 사용할 수 있습니다.

**실행 방법 (두 가지)**

```
방법 1 — 설치 폴더 안의 uninstall.bat 실행 (기본값이 자동 설정)
  C:\FileCleaner\uninstall.bat  더블클릭

방법 2 — 배포패키지의 uninstall.bat 실행 후 경로 직접 입력
  배포패키지\uninstall.bat  더블클릭
  → "설치 폴더 [...]:" 프롬프트에 실제 설치 경로 입력
    예) C:\ProgramData\FileCleaner
```

실행 시 화면:

```
제거할 설치 폴더를 입력하세요.
(다른 경로에 설치된 경우 해당 경로를 직접 입력하세요)

설치 폴더 [C:\배포패키지]: C:\ProgramData\FileCleaner
```

자동으로 처리되는 항목:

| 순서 | 항목 | 설명 |
|---|---|---|
| 1 | 작업 스케줄러 삭제 | `File_Auto_Cleaner` 등록 제거 |
| 2 | 격리 폴더 삭제 | `config.json`에서 경로를 자동 감지, 삭제 여부를 별도로 확인 |
| 3 | 프로그램 파일 삭제 | `file_cleaner.*`, `config.json`, `run_cleaner*.bat`, `activity_log.txt`, `사용법.*` |
| 4 | Portable Python 삭제 | `python\` 폴더 (Portable 버전인 경우) |
| 5 | 설치 폴더 정리 | 파일 삭제 후 빈 폴더이면 자동 제거 |

> `uninstall.bat` 은 모든 작업을 실행 전에 목록으로 표시하고 **"y"를 입력해야만** 제거를 시작합니다. 실수로 실행해도 Enter 만 누르면 취소됩니다.

---

### 수동 제거

`uninstall.bat` 이 없거나 직접 처리하고 싶은 경우:

**1.** 작업 스케줄러에서 등록 제거:

`Win + S` → **작업 스케줄러** → 작업 스케줄러 라이브러리
→ `File_Auto_Cleaner` 우클릭 → **삭제**

**2.** 프로그램 파일 삭제:

설치 폴더의 `file_cleaner.py`(또는 `.exe`), `config.json`, `run_cleaner.bat`, `activity_log.txt`, `uninstall.bat` 파일을 탐색기에서 삭제합니다.

Portable 버전인 경우 `python\` 폴더도 함께 삭제합니다.

> 격리 폴더(`quarantine_dir`에 설정한 경로)와 그 안의 파일은 별도로 직접 확인 후 삭제하세요.

## 동작 방식

```
[실행]
  ↓
1단계 — 격리
  대상 폴더를 탐색하여 필터(확장자, 키워드)에 맞는 파일 중
  생성일 기준 quarantine_days를 초과한 파일을 Quarantine 폴더로 이동

  ↓

2단계 — 삭제
  Quarantine 폴더 내 파일 중
  이동 날짜 기준 delete_days를 초과한 파일을 영구 삭제
```

**시스템 보호 경로는 절대 건드리지 않습니다:**

`C:\Windows`, `C:\Program Files`, `C:\ProgramData`, `C:\Recovery`,
`%APPDATA%` (AppData\Roaming), `%LOCALAPPDATA%` (AppData\Local) 등

보호 경로는 코드에 고정된 목록과 실행 환경의 환경변수를 합산해 자동 구성됩니다.
`config.json`의 `exclude_paths`에 경로를 추가하면 보호 목록이 확장됩니다.

읽기 전용(R) 또는 시스템(S) 속성 파일도 건너뜁니다.

## 즉시 테스트 방법

설치 후 실제 격리·삭제가 동작하는지 테스트 파일로 바로 확인하는 방법입니다.
기준일을 임시로 `0`으로 낮춰서 파일 생성 즉시 격리·삭제가 발생하게 만듭니다.

### 1단계 — 테스트 파일 만들기

`Win + R` → 아래 명령어 입력 → Enter:

```
cmd /k echo test > "%USERPROFILE%\Downloads\test_file.tmp"
```

`C:\Users\사용자명\Downloads\` 에 `test_file.tmp` 파일이 생성됩니다.

### 2단계 — config.json 임시 수정

`config.json`을 메모장으로 열어 아래 두 값을 `0`으로 변경합니다.

```json
"quarantine_days": 0,
"delete_days":     0,
```

저장 후 메모장을 닫습니다.

### 3단계 — 수동 실행

**Python 버전:**

`Win + R` → 아래 입력 → Enter:

```
cmd /k python "C:\FileCleaner\file_cleaner.py" --config
```

**EXE 배포 버전 (`setup.bat`으로 설치한 경우):**

```
cmd /k "C:\FileCleaner\file_cleaner.exe" --config
```

### 4단계 — 결과 확인

**격리 확인 (1회 실행 후)**

```
INFO  [격리]  C:\Users\사용자명\Downloads\test_file.tmp
               →  C:\Users\사용자명\Downloads\Quarantine\test_file.tmp  (생성 후 0일)
INFO  [격리 소계]  1개 파일 이동
INFO  [삭제 소계]  0개 파일 영구 삭제
```

탐색기에서 `C:\Users\사용자명\Downloads\Quarantine\` 폴더를 열면 `test_file.tmp`가 이동된 것을 확인할 수 있습니다.

**삭제 확인 (한 번 더 실행)**

`delete_days`도 `0`이므로 두 번째 실행에서 격리 파일이 바로 영구 삭제됩니다.

```
INFO  [격리 소계]  0개 파일 이동

## 주의 사항

- **격리 폴더를 주기적으로 확인**하세요. 필요한 파일이 실수로 들어갔을 경우 `delete_days` 기간 내에 꺼내야 합니다.

- 프로그램을 처음 사용할 때 `quarantine_days`를 넉넉하게 (예: 60~90일) 설정하고, 격리 결과를 며칠 관찰한 뒤 기준을 좁혀가는 것이 안전합니다.

### 코드 수준 취약점 점검

#### 경로 조작(Path Traversal) — 방어됨

`config.json`의 `target_path`와 `quarantine_dir`는 `os.path.normpath()`로 정규화한 뒤 `PROTECTED_PATHS`와 접두사 비교합니다. 공격자가 `../` 등을 삽입해도 정규화 이후 시스템 경로 진입을 차단합니다.

```
입력:  C:\Users\work\..\..\..\Windows\System32
정규화: C:\Windows\System32   →  PROTECTED_PATHS 매칭 → 거부
```

#### 커맨드 인젝션(Command Injection) — 해당 없음

`subprocess`, `os.system`, `eval`, `exec` 등 셸 명령 실행 함수를 사용하지 않습니다. 파일 조작은 전부 Python 내장 API(`shutil.move`, `os.remove`)로 처리합니다.

#### 임의 코드 실행 — 해당 없음

설정 파일을 `json.load()`로만 읽습니다. `pickle`, `yaml.load()`, `eval()` 등 역직렬화 취약점이 발생하는 포맷을 사용하지 않습니다.

#### 심볼릭 링크(Symlink) 공격 — 부분 주의 필요

`os.walk`와 `shutil.move`는 기본적으로 심볼릭 링크를 따라갑니다.

| 위험 시나리오 | 발생 조건 |
|---|---|
| 공격자가 대상 폴더 내에 시스템 경로를 가리키는 심볼릭 링크를 생성한 경우 | 공격자가 대상 폴더에 쓰기 권한을 가진 경우에만 가능 |

**완화 방법:** 대상 폴더를 신뢰할 수 있는 경로(개인 작업 폴더)로만 지정하고, 불특정 다수가 쓰기 권한을 가진 공유 폴더를 대상으로 하지 마세요.

#### 로그 인젝션(Log Injection) — 구조적으로 제한됨

`activity_log.txt`에는 파일 경로와 경과 일수만 기록됩니다. 파일명에 개행 문자(`\n`)가 포함된 경우 로그 가독성이 깨질 수 있으나, 로그 파일 자체가 실행되거나 파싱되는 구조가 아니므로 보안 위협으로 이어지지 않습니다.

## 폐쇄망 · Python 미설치 PC 배포 가이드

인터넷이 연결되지 않는 폐쇄망 환경, 또는 Python 이 설치되어 있지 않은 PC 에 배포할 때 사용하는 방법입니다.
핵심은 **배포자가 미리 Python 런타임까지 포함된 단독 실행 파일(.exe)을 만들어 전달**하는 것입니다.
수신자 PC 에는 Python 설치도, 인터넷 연결도 필요하지 않습니다.

---

### 역할 구분

| 역할 | 대상 | 해야 할 일 |
|---|---|---|
| **배포자** | 프로그램을 만들어 전달하는 사람 (인터넷 PC, Python 설치됨) | `build.bat` 실행 → ZIP 전달 |
| **수신자** | 프로그램을 설치해서 쓰는 사람 (폐쇄망 PC, Python 없어도 됨) | ZIP 압축 해제 → `setup.bat` 실행 |

---

### 배포 파일 준비 방법

**방법 A — GitHub 에서 내려받기 (권장)**

https://github.com/Sdapaul/delete_pii 에서 `release/` 폴더의 파일을 내려받으면 `setup.bat`, `run_cleaner.bat`, `배포방법.md` 가 이미 포함되어 있습니다.
`build.bat` 만 추가로 실행해 `file_cleaner.exe` 를 생성하면 배포 패키지가 완성됩니다.

**방법 B — 소스코드에서 직접 빌드**

아래 절차대로 `build.bat` 을 실행합니다.

---

### 배포자가 할 일 (인터넷 PC 에서, 1회만)

**1.** 프로젝트 폴더에서 `build.bat` 을 더블클릭합니다.

자동으로 처리되는 작업:
- PyInstaller 설치 (없는 경우 자동 설치)
- `file_cleaner.exe` 빌드 (Python 런타임 포함, 30초~1분 소요)
- `배포패키지\` 폴더 자동 조립

**2.** 빌드 완료 후 `배포패키지\` 폴더가 생성됩니다.

```
배포패키지\
  ├── file_cleaner.exe   ← Python 포함된 단독 실행 파일
  ├── run_cleaner.bat    ← 스케줄러 실행 래퍼
  ├── setup.bat          ← 수신자용 자동 설치 스크립트
  └── 사용법.md           ← 참고 가이드
```

**3.** `배포패키지\` 폴더를 우클릭 → **보내기 → 압축(ZIP) 폴더** 로 압축합니다.

**4.** 압축된 ZIP 파일을 USB 또는 사내 파일 서버를 통해 수신자에게 전달합니다.

> `config.json` 은 배포 파일에 포함하지 마세요. 개인 경로 정보가 담겨 있고, 수신자 PC 에서 setup.bat 이 새로 생성합니다.

---

### 수신자가 할 일 (폐쇄망 PC 에서)

**1.** 전달받은 ZIP 파일을 우클릭 → **모두 압축 풀기** 로 압축 해제합니다.

**2.** 압축 해제된 폴더 안의 `setup.bat` 을 더블클릭합니다.

자동으로 처리되는 작업:
- 프로그램 파일을 지정한 폴더(기본: `C:\FileCleaner\`)에 복사
- 대화형 초기 설정 (검사 폴더, 격리 기간 등 입력)
- Windows 작업 스케줄러 자동 등록

**3.** 설치 완료 후에는 매일 지정한 시각에 자동 실행됩니다.

### 문제 1 — Windows SmartScreen 경고 대처

인터넷에서 내려받지 않은 파일이라도, **서명되지 않은 exe** 를 처음 실행할 때 아래와 같은 경고창이 나타날 수 있습니다.

```
Windows가 PC를 보호했습니다.
Microsoft SmartScreen이 알 수 없는 앱의 시작을 차단했습니다.
이 앱을 실행하면 PC가 위험해질 수 있습니다.
```

이 경고는 프로그램이 위험해서가 아니라, Windows 에 등록되지 않은 새 프로그램이기 때문에 나타납니다.

**해제 방법:**

**1.** 경고창 하단의 **"추가 정보"** 를 클릭합니다.

```
[경고창 예시]
┌──────────────────────────────────────────┐
│ Windows가 PC를 보호했습니다.             │
│                                          │
│ 앱: setup.bat 또는 file_cleaner.exe      │
│ 게시자: 알 수 없음                       │
│                                          │
│  → "추가 정보" 클릭  ←                   │
│                                          │
│         [실행 안 함]                     │
└──────────────────────────────────────────┘
```

**2.** "추가 정보" 클릭 후 화면이 바뀌면 **"실행"** 버튼이 나타납니다.

```
[추가 정보 클릭 후]
┌──────────────────────────────────────────┐
│ Windows가 PC를 보호했습니다.             │
│                                          │
│ 앱: file_cleaner.exe                     │
│ 게시자: 알 수 없음                       │
│                                          │
│  → "실행" 클릭  ←                        │
│                                          │
│  [실행]          [실행 안 함]            │
└──────────────────────────────────────────┘
```

**3.** "실행" 을 클릭하면 프로그램이 정상 실행됩니다.

> 이 과정은 **최초 1회만** 필요합니다. 이후 실행부터는 경고창이 나타나지 않습니다.

**ZIP 파일 자체에서 차단되는 경우 (인터넷 경유 전달 시)**

ZIP 을 이메일이나 웹에서 내려받은 경우, Windows 가 ZIP 자체를 "인터넷에서 온 파일" 로 표시합니다.
이때는 압축 해제 전에 먼저 차단을 해제해야 합니다.

**1.** ZIP 파일을 우클릭 → **속성** 클릭

**2.** 속성 창 하단에 아래 문구가 있으면:

```
보안: 이 파일은 다른 컴퓨터에서 가져온 것으로...
      [√] 차단 해제  ← 이 체크박스에 체크
```

**3.** **"차단 해제"** 체크박스를 체크하고 **확인** 클릭

**4.** 이후 ZIP 을 압축 해제하면 내부 파일들도 차단 없이 실행됩니다.

### 문제 2 — 백신이 exe 를 차단하는 경우

PyInstaller 로 만든 exe 는 실행 시 임시 폴더에 자신을 압축 해제하는 구조라서, 일부 백신이 오탐(false positive)으로 차단할 수 있습니다.

**대처 방법 (택 1):**

**방법 A — 백신 예외 폴더 등록 (권장)**

백신 관리자 화면에서 `C:\FileCleaner\` 폴더를 검사 예외로 추가합니다.
백신 제품마다 위치가 다르지만 일반적으로 **설정 → 예외 항목 → 폴더 추가** 경로에 있습니다.

**방법 B — 폴더 방식(--onedir)으로 재빌드**

`build.bat` 안의 pyinstaller 명령을 아래와 같이 수정합니다.

```bat
:: 기존 (파일 1개 방식)
pyinstaller --onefile --name file_cleaner ...

:: 변경 (폴더 방식, TEMP 폴더 의존 없음)
pyinstaller --onedir --name file_cleaner ...
```

폴더 방식으로 빌드하면 `dist\file_cleaner\` 폴더 안에 exe 와 의존 파일들이 분리되어 생성됩니다.
백신 오탐이 크게 줄고, 임시 폴더를 사용하지 않아 보안 정책이 엄격한 PC 에서도 잘 동작합니다.

**폴더 방식 배포 패키지 구성:**

```
배포패키지\
  ├── file_cleaner\          ← 이 폴더 전체를 반드시 함께 배포
  │     ├── file_cleaner.exe ← 실행 파일 (이것만으로는 실행 안 됨)
  │     ├── python3XX.dll
  │     └── ... (의존 파일들, 건드리지 않아도 됨)
  ├── run_cleaner.bat
  ├── setup.bat
  └── 사용법.md
```

수신자는 동일하게 `setup.bat` 을 실행하면 됩니다.
단, 배포할 때 `file_cleaner\` **폴더 전체**가 빠지지 않도록 주의하세요.
`file_cleaner.exe` 파일 하나만 복사하면 실행되지 않습니다.

> 어느 방식이 맞는지 모르겠다면 **방법 A(백신 예외 등록)** 를 먼저 시도하세요.
> 폴더 방식은 배포가 약간 복잡해지는 대신 보안 환경 적합성이 높습니다.

---

### 폐쇄망 배포 흐름 요약

```
[배포자 — 인터넷 PC, Python 있음]
  1. build.bat 실행
  2. 배포패키지\ 폴더 생성됨
  3. ZIP 으로 압축
  4. USB 또는 사내 파일 서버로 전달
          ↓
[수신자 — 폐쇄망 PC, Python 없어도 됨]
  5. ZIP 압축 해제
  6. setup.bat 더블클릭
  7. 설치 완료 → 매일 자동 실행
```

## 금융사 내부망 — EXE 차단 환경 배포 방법

보안 정책이 엄격한 금융사 내부망에서는 서명되지 않은 `.exe` 실행이 차단될 수 있습니다.
이 경우 아래 두 방법 중 하나를 선택합니다.

| 방법 | 조건 | 특징 |
|---|---|---|
| **방법 1 — Python 스크립트 직접 실행** | PC 에 Python 이 설치되어 있는 경우 | 가장 단순. exe 불필요 |
| **방법 2 — Portable Python 패키지** | Python 미설치, exe 차단 환경 | Python 폴더를 함께 전달. 설치 없이 실행 |

---

### 방법 1 — Python 스크립트 직접 실행

PC 에 Python 3.8 이상이 설치되어 있다면 `.py` + `.bat` 만으로 동작합니다.
`file_cleaner.exe` 는 필요 없습니다.

#### 배포 파일 구성 (3개)

```
배포패키지\
  ├── file_cleaner.py   ← 메인 프로그램 (스크립트)
  ├── run_cleaner.bat   ← 스케줄러 실행 래퍼
  └── 사용법.md
```

`config.json` 은 포함하지 않습니다. 수신자 PC 에서 최초 실행 시 자동 생성됩니다.

#### 수신자 설치 절차

**1.** 배포받은 폴더를 원하는 위치에 복사합니다. (예: `C:\FileCleaner\`)

**2.** `run_cleaner.bat` 을 더블클릭합니다. 최초 실행 시 대화형으로 설정을 입력합니다.

**3.** 작업 스케줄러에 등록합니다. (사용법.md STEP 5 참고)

#### run_cleaner.bat 동작 방식

```bat
python "%~dp0file_cleaner.py" --config
```

`python` 명령이 PATH 에 등록된 Python 을 그대로 사용합니다.
PC 에서 `cmd → python --version` 으로 확인 가능합니다.

---

### 방법 2 — Portable Python 패키지 (설치 없이 실행)

Python 이 설치되어 있지 않고 exe 실행도 차단된 경우에 사용합니다.
Python 공식 배포 **Embeddable Package**(zip)를 함께 전달해 설치 없이 실행합니다.

> **핵심 원리:** Python Embeddable Package 는 ZIP 파일을 압축 해제만 하면 되는 폴더 구조입니다. 레지스트리나 PATH 수정이 없고, `python.exe` 를 절대 경로로 직접 호출합니다.

#### 배포 파일 구성

```
배포패키지_portable\
  ├── python\                   ← Portable Python (압축 해제된 폴더)
  │   ├── python.exe            ← Python 런타임 (설치 불필요)
  │   ├── python313.zip
  │   └── ... (표준 라이브러리 등)
  ├── file_cleaner.py           ← 메인 프로그램
  ├── setup.bat                 ← 수신자용 자동 설치 스크립트
  ├── run_cleaner_portable.bat  ← 스케줄러 실행 래퍼
  └── 사용법.md / 사용법.pdf
```

#### 배포자가 할 일 (인터넷 PC 에서, 1회만)

**1.** `release\build_portable.bat` 을 실행하기 전에 Portable Python 을 준비합니다.

`Win + R` → 브라우저에서 아래 주소로 이동:

```
https://www.python.org/downloads/windows/
```

페이지에서 최신 안정 버전의 **"Windows embeddable package (64-bit)"** 를 클릭해 ZIP 을 내려받습니다.

```
예시: python-3.13.4-embed-amd64.zip  (약 11 MB)
```

**2.** 내려받은 ZIP 을 `release\python\` 폴더에 압축 해제합니다.

```
release\
  └── python\
        ├── python.exe    ← 이 파일이 있어야 합니다
        └── ...
```

**3.** `release\build_portable.bat` 을 더블클릭합니다.

자동으로 처리되는 작업:
- `python\` 폴더 포함 여부 확인
- `배포패키지_portable\` 폴더 자동 조립

**4.** 생성된 `배포패키지_portable\` 폴더를 ZIP 으로 압축해서 전달합니다.

> `config.json` 은 포함하지 마세요. 수신자 PC 에서 `setup.bat` 이 새로 생성합니다.

#### 수신자가 할 일 (내부망 PC 에서)

**1.** 전달받은 ZIP 을 압축 해제합니다.

**2.** `setup.bat` 을 더블클릭합니다.

자동으로 처리되는 작업:
- `python\` 폴더 포함 여부 확인
- 설치 폴더(`C:\FileCleaner\` 기본) 에 파일 복사
- 대화형 초기 설정
- 작업 스케줄러 자동 등록

**3.** 설치 완료 후 매일 자정 설정한 시각에 자동 실행됩니다.

#### run_cleaner_portable.bat 동작 방식

```bat
"%~dp0python\python.exe" "%~dp0file_cleaner.py" --config
```

`%~dp0python\python.exe` 는 bat 파일이 있는 폴더 기준 상대 경로입니다.
PATH 나 시스템 Python 과 무관하게 동작합니다.

#### Portable Python 버전 선택 기준

| 항목 | 권장 선택 |
|---|---|
| 아키텍처 | **64-bit** (`embed-amd64`) — 최근 PC 대부분 해당 |
| 32-bit PC 인 경우 | `embed-win32` 버전 선택 |
| Python 버전 | 최신 안정 버전 (3.11 이상 권장) |

> **이 프로그램은 표준 라이브러리만 사용**하므로 어떤 버전의 Embeddable Package 를 사용해도 정상 동작합니다.

### 방법 선택 요약

```
[PC 환경 확인]
  │
  ├─ Python 이 설치되어 있다 (python --version 출력 됨)
  │    └─ → 방법 1 사용 (file_cleaner.py + run_cleaner.bat)
  │
  └─ Python 이 없다 또는 설치 불가
       │
       ├─ EXE 실행 가능 (SmartScreen 경고 허용)
       │    └─ → 기존 EXE 배포 방법 사용 (build.bat → setup.bat)
       │
       └─ EXE 도 차단됨 (금융사 내부망 등)
            └─ → 방법 2 사용 (Portable Python + setup_portable.bat)
