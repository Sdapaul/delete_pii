# 배포 방법 가이드

이 폴더는 **EXE 배포 패키지** 를 조립하는 데 필요한 스크립트를 포함합니다.
수신자 PC 에 Python 이 없어도 설치·운용할 수 있도록 구성합니다.

---

## 파일 구성

| 파일 | 역할 |
|---|---|
| `setup.bat` | 수신자용 자동 설치 스크립트 |
| `run_cleaner.bat` | 작업 스케줄러 실행 래퍼 (EXE 버전) |
| `배포방법.md` | 이 파일 |

> `file_cleaner.exe` 는 이 폴더에 포함되어 있지 않습니다.
> 아래 절차에 따라 빌드 후 추가하세요.

---

## 배포 절차

### 역할 구분

| 역할 | 대상 | 해야 할 일 |
|---|---|---|
| **배포자** | 인터넷 PC, Python 설치됨 | `build.bat` 실행 → ZIP 전달 |
| **수신자** | 폐쇄망 PC, Python 없어도 됨 | ZIP 해제 → `setup.bat` 실행 |

---

### STEP 1 — 소스코드 준비 (배포자)

GitHub 저장소에서 클론하거나 ZIP 으로 내려받습니다.

```
https://github.com/Sdapaul/delete_pii
```

또는 이미 로컬에 소스코드가 있다면 해당 폴더를 사용합니다.

---

### STEP 2 — EXE 빌드 (배포자)

소스코드 루트 폴더에서 `build.bat` 을 더블클릭합니다.

자동으로 처리되는 작업:
- PyInstaller 설치 (없는 경우 자동 설치)
- `file_cleaner.exe` 빌드 (Python 런타임 포함, 30초~1분 소요)
- `배포패키지\` 폴더 자동 조립

빌드 완료 후 `배포패키지\` 폴더가 생성됩니다:

```
배포패키지\
  ├── file_cleaner.exe   ← Python 포함된 단독 실행 파일
  ├── run_cleaner.bat    ← 스케줄러 실행 래퍼
  ├── setup.bat          ← 수신자용 자동 설치 스크립트
  └── 사용법.md           ← 참고 가이드
```

---

### STEP 3 — 수동 조립 (release/ 폴더 기준, build.bat 미사용 시)

`release/` 폴더를 사용해 수동으로 배포 패키지를 만들 수 있습니다.

1. `release/` 폴더를 복사해 `배포패키지/` 폴더를 만듭니다.
2. 빌드된 `file_cleaner.exe` 를 `배포패키지/` 에 추가합니다.
3. `사용법.md` 와 `사용법.pdf` 를 `배포패키지/` 에 추가합니다.

최종 배포 패키지 구성:

```
배포패키지\
  ├── file_cleaner.exe   ← STEP 2 에서 빌드
  ├── run_cleaner.bat    ← release\run_cleaner.bat
  ├── setup.bat          ← release\setup.bat
  ├── 사용법.md           ← 루트의 사용법.md
  └── 사용법.pdf          ← 루트의 사용법.pdf
```

---

### STEP 4 — 전달 (배포자)

`배포패키지\` 폴더를 ZIP 으로 압축 후 전달합니다.

```
배포패키지\ 폴더 우클릭 → 보내기 → 압축(ZIP) 폴더
```

USB 또는 사내 파일 서버로 수신자에게 전달합니다.

> **주의:** `config.json` 은 포함하지 마세요.
> 개인 경로 정보가 담겨 있고, 수신자 PC 에서 `setup.bat` 이 새로 생성합니다.

---

### STEP 5 — 설치 (수신자, 폐쇄망 PC)

1. 전달받은 ZIP 을 우클릭 → **모두 압축 풀기**
2. 압축 해제된 폴더 안의 `setup.bat` 더블클릭
3. 자동 처리:
   - 프로그램 파일을 지정한 폴더(`C:\FileCleaner\` 기본)에 복사
   - 대화형 초기 설정 (검사 폴더, 격리 기간 등)
   - Windows 작업 스케줄러 자동 등록

---

## 주의 사항

### SmartScreen 경고

서명되지 않은 exe 를 처음 실행할 때 경고창이 나타날 수 있습니다.

- 경고창에서 **"추가 정보"** 클릭 → **"실행"** 클릭 (최초 1회만)
- 이메일/웹으로 전달한 ZIP 은 압축 해제 전에 ZIP 파일 속성에서 **"차단 해제"** 체크

### 백신 오탐

PyInstaller EXE 가 임시 폴더에 압축 해제되는 구조라 일부 백신이 오탐할 수 있습니다.

- **권장:** 백신 설정에서 `C:\FileCleaner\` 폴더를 검사 예외로 추가
- **대안:** `build.bat` 의 `--onefile` 을 `--onedir` 로 변경 후 재빌드

---

## 업데이트 배포

프로그램이 업데이트된 경우 수신자 PC 를 갱신하는 방법입니다.

1. 배포자: 최신 소스코드로 `build.bat` 재실행 → 새 `file_cleaner.exe` 생성
2. 새 `file_cleaner.exe` 를 수신자에게 전달
3. 수신자: 기존 설치 폴더(`C:\FileCleaner\`)의 `file_cleaner.exe` 를 새 파일로 교체

> `config.json` 은 교체 불필요. 기존 설정이 그대로 유지됩니다.
> 작업 스케줄러도 재등록 불필요.
